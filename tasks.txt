-----------------
BASICS
-----------------
1. Найдите корабли водоизмещением более 30000т
2. Перечислите названия, водоизмещение и число орудий кораблей, участвовавших в сражении при Гвадалканале.
3. Перечислите все корабли, упомянутые в БД (корабли упомянуты в таблицах Ships, Outcomes).
4. Укажите страны, имеющие и линкоры и крейсеры.
5. Укажите корабли, которые были повреждены в одном сражении и позднее участвовали в другом сражении.
6. Укажите названия кораблей и сражения, в которых участвовали по крайней мере 2 корабля одной страны. При необходимости введите в базу данных корабли, удовлетворяющие этому условию

-----------------
EXISTS_IN_ALL_ANY
-----------------
1. Найдите страны, корабли которых имеют наибольшее число орудий.
2. Найдите классы кораблей, в которых хотя бы один корабль был потоплен в сражении.
3. Укажите сражения, в которых участвовали корабли с орудиями калибра более 10 дюймов.
4. Найдите названия и даты сражений, в которых участвовали корабли класса Конго.
5. Найдите названия кораблей, имеющих наибольшее число орудий среди всех кораблей такого же водоизмещения.

-----------------
AGGREGATE
-----------------
1. Получите названия кораблей, упомянутых в таблицах Outcomes и Ships, которые являются первыми кораблями своего класса ( название класса совпадает с названием первого спущенного на воду корабля)
2. Укажите названия кораблей, которые построены после того как был построен первый корабль соответствующего класса.
3. Перечислите все классы кораблей, упомянутые в таблицах Classes и Ships базы данных.
4. Укажите названия кораблей, которые не упоминаются в Outcomes как корабли, участвовавшие в сражениях.
5. Укажите классы, для которых в таблице Ships отсутствуют сведения о кораблях, спущенных на воду.

-----------------
GROUPBY_VIEWS
-----------------
1. Определите число классов линейных кораблей.
2. Определите среднее число орудий для классов линейных кораблей.
3. Определите средний калибр орудий для классов, имеющих число орудий, больше среднего числа орудий для всех классов.
4. Для каждого класса определите год, когда был спущен на воду первый корабль этого класса.
5. Для каждого класса определите число кораблей этого класса, потопленных в сражении.
6. Для каждого класса, содержащего не менее трех кораблей, определите число кораблей этого класса, потопленных в сражении.
7. Определите средний вес снарядов для кораблей каждой страны. Вес снаряда примерно равен половине куба его калибра.
8. Постройте представление BritishShips, содержащее название корабля Великобритании, его класс, тип, количество и калибр орудий, водоизмещение и год спуска на воду. 
9. Имея представление BritishShips, вставьте в него новый кортеж и сделайте запрос о среднем количестве  орудий британских линкоров, спущенных на воду в каждом году.

-----------------
JOINS_UPDATES
-----------------
1. Измените таблицу Classes так, чтобы калибры их орудий измерялись в сантиметрах (1 дюйм=2.5 см).
2. Увеличьте число орудий для классов кораблей Великобритании и США на 1 и уменьшите их калибр на 2.
3. Измените таблицу Battles добавьте столбец ShipNum и введите число кораблей, которые участвовали в каждом сражении.
4. Найдите названия всех кораблей водоизмещением более 36000 тонн.
5. Запросите всю информацию о кораблях и классах, причем требуется вся информация, даже если нет сведений о кораблях класса или нет сведений о классе некоторых кораблей. Выполните тот же запрос при условии, что, не нужна информация о классе,  если нет сведений кораблях этого класса. Выполните тот же запрос при условии, что, не нужна информация о кораблях,  если нет сведений классах этих кораблей (3 запроса).

-----------------
TRIGGERS
-----------------
1. Создайте систему аудита для таблицы Ships с использованием операторного триггера. При любом действии с таблицей Ships в таблицу аудита следует записать идентификатор пользователя, дату , время и имя триггера. Используйте тип TIMESTAMP для времени. Чем отличается операторный и строковый триггер?
2. Создайте систему аудита для таблицы Сlasses с использованием строкового триггера. При любом действии с таблицей Сlasses в таблицу аудита следует записать имя пользователя, дату , время и имя триггера. Продемонстрируйте, чем отличается операторный и строковый триггер.
3. При удалении информации о сражении (строки в таблице Battles) должны быть удалены соответствующие строки из таблицы Outcomes,  которые ссылаются на  удаленные сражения. Используйте псевдозаписи :old, :new.
4. Создайте систему аудита таблицы Outcomes . В таблицу аудита запишите  имя пользователя, дату и время, тип действия, историю (старые и новые названия кораблей). Используйте предикаты IF-THEN-ENDIF. 
5. Создайте таблицу новых кораблей (год спуска на воду >1950).  Если в таблицу Ships оператор вводит сведения о новом корабле, то эти сведения следует записать в новую таблицу. Используйте условие WHEN. 

-----------------
CURSORS
-----------------
1. Создайте явный курсор для выбора названия корабля и сражения, в котором он участвовал. Используйте простой цикл (LOOP  EXIT - WHEN ) для извлечения данных из курсора и параметры курсора (%ROWCOUNT,  %NOTFOUND).
2. Создайте явный курсор для выбора названия корабля и класса. Используйте тип данных  - записи для переменных связи и курсора. Простой цикл для получения данных из курсора (LOOP  EXIT - WHEN ) 
3. Создайте явный курсор для выбора характеристик кораблей, спущенных на воду до 1943 года. Используйте тип данных – записи и цикл For для получения данных из курсора (FOR - IN – LOOP - END LOOP). 
4. Создайте явный курсор с параметрами для выбора названия корабля, сражения и даты спуска на воду для  кораблей, для которых  результат сражения, задан в параметрах курсора (‘ok’/‘sunk’ /‘damaged’). Используйте тип данных – записи и цикл для получения данных из курсора.
5. Создайте курсор с подзапросом для выбора названия корабля, класса и страны. Используйте тип данных – записи и цикл For  для получения данных из курсора.
